## User Story

Como **visitante alinhado com a persona "Marcelo"** vindo de anúncio Instagram ou busca Google, quero ler o Hero em até 2.5s, reconhecer minha realidade nas Seções 1-3 e ter o **primeiro CTA WhatsApp visível e funcional**, para que eu sinta que vale a pena descer pelo resto da landing — e para que a marca ganhe a primeira oportunidade de conversão.

## Dev Notes

- **CON-013 + FR-050:** copy é **literal** de `05-copy-landing.md` versão 2026-05-05 + correções `copy-audit.md` §7. Validar visualmente que NÃO há "garantia/garantir" em nenhum trecho destas 3 seções (CON-013 checklist). Não inventar variações de copy.
- **Persona "Marcelo" NÃO aparece:** persona-avatar é interna (CON-013). Nenhum nome de persona pode aparecer nas seções 2 ou 3 (na 1.4 também não).
- **ADR-003 (GSAP SSR):** `'use client'` em todas as seções animadas. Estado inicial via CSS `opacity: 0` para evitar flicker pré-hidratação. `ScrollTrigger.refresh()` em `requestAnimationFrame` no `<AnimationsProvider>` (já feito na 1.2) — mas validar que continua funcionando após adicionar 3 seções animadas. Mitigação Risco R1.
- **ADR-004 (foto Hero):** `priority + fetchPriority="high"` é mandatório no Hero. Foto SEM esses atributos = LCP estoura. Validar via Network panel que foto é fetched antes do JS bundle.
- **Risco R2 (CLS fontes):** confirmar que `next/font` da 1.2 não regrediu — Lighthouse CLS <0.1 obrigatório.
- **Sticky nav iOS:** `position: sticky` tem histórico de bug em iOS Safari. Testar em iPhone real (QG 9.3); fallback `position: fixed` aceitável se necessário, mas validar primeiro.
- **Counter tween:** usar `gsap.to({ value: 0 }, { value: 1200, duration: 1.2, onUpdate: () => setText(Math.round(this.value)) })`. Em `prefers-reduced-motion`, mostrar valor final direto (sem animação).
- **Curve divider mobile:** clip-path scrub funciona bem desktop, mas em mobile pode causar paint spikes. Usar SVG estático mobile (`gsap.matchMedia` com isMobile/isDesktop branches).
