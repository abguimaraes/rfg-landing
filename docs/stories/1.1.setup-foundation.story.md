## User Story

Como **desenvolvedor da RFG Landing**, quero um repositório novo `abguimaraes/rfg-landing` com Next.js 15 (App Router), TypeScript strict, Tailwind + tokens RFG, ESLint flat config, Prettier, pnpm, GitHub Actions e Vercel preview funcionando, para que **as Stories 1.2-1.8 partam de uma fundação estável, deterministica e isolada do PostFeito** (CON-011).

---

## Context

Esta é a story de abertura do epic. Não há código pré-existente — o repo é 100% novo (CON-011 — codebase isolado, zero importação do PostFeito). O resultado da story é um boilerplate Next.js 15 deployando em Vercel preview com a página inicial em estado "Em construção" (placeholder), Lighthouse já passando ≥95 baseline, tokens da paleta RFG carregados via `tokens.css` drop-in (`design-system.md` §9), Tailwind config com cores `rfg.dark/mid/light` + neutros tingidos azul (FR-052), `next/font/google` Manrope+Inter pré-configurado mas com uso real só na 1.2, e CI rodando lint+typecheck+build em PRs.

O domínio `www.rfgcorretora.com.br` **não é apontado nesta story** — fica para a 1.8 (DNS + go-live). Aqui, Vercel gera URL preview própria (ex.: `rfg-landing-abguimaraes.vercel.app`) e é nela que validamos a fundação.

Referências obrigatórias: `architecture.md` ADRs 001/004/006, `design-system.md` §9-10, PRD Story 1.1 §8, FR-052, NFR-024/025.

## Dev Notes

- **ADR-001 (Next 15 + RSC):** App Router obrigatório. Componentes interativos só serão `'use client'` quando precisarem (Story 1.2 em diante). Nesta story, `page.tsx` placeholder é Server Component.
- **ADR-004 (next/image):** já habilitar `formats: ['image/avif', 'image/webp']` em `next.config.ts` para que as próximas stories já partam com AVIF on.
- **ADR-006 (a11y):** ESLint `jsx-a11y` zero warnings é gate de CI. Skip-link já adicionado nesta story.
- **CON-009 (repo privado):** `abguimaraes/rfg-landing` privado. Branch protection inegociável (NFR-024).
- **CON-010 (Vercel mesma conta):** mesma conta dos outros projetos Anderson, mas projeto isolado (sem mistura com PostFeito).
- **CON-011 (Codebase 100% novo):** zero import do PostFeito. Mesmo o `tailwind.config.ts` é escrito do zero a partir do `design-system.md`, não copiado.
- **FR-051 / FR-052 / Risco R2 (CLS fontes):** `next/font/google` com `adjustFontFallback: true` é a mitigação canonical do FOUT/FOIT. Validar em Lighthouse — qualquer CLS >0.1 nesta story aborta.
- **Risco R5 (preview vs prod):** GA4/Pixel ainda não estão integrados nesta story (Story 1.2), mas já documentar em README que `process.env.VERCEL_ENV === 'production'` é o gate quando chegar a hora.
- **Não executar `git push --force` em hipótese alguma** — CON-009 + AGENT-AUTHORITY (push é exclusivo de @devops; nesta story de setup, @devops cria o repo e abre PR conforme protocolo).
