## User Story

Como **mecanismo de busca (Google) ou bot social (Facebook/LinkedIn/WhatsApp Preview)** indexando ou compartilhando a landing, quero **metadata completa, Open Graph dinâmico, JSON-LD `LocalBusiness` + `InsuranceAgency` + `FAQPage`, sitemap, robots e ícones** corretos; e como **visitante LGPD-aware**, quero **páginas `/privacidade` e `/termos` válidas + ConsentBanner que gateia Meta Pixel até consent explícito**, para que **a landing rankeie no SEO local "corretora de seguros Maceió", apareça com rich snippets, gere OG cards bonitos e respeite LGPD em produção**.

---

## Context

Story de instrumentação não-visual (mas crítica para **SEO Lighthouse 100** — NFR-004 — e **conformidade LGPD** — CON-004). Implementa **Metadata API completa**, **OG dinâmico via `next/og`**, **JSON-LD com 2 schemas combinados** (LocalBusiness + InsuranceAgency em layout root + FAQPage no FaqSection), **`robots.ts` + `sitemap.ts` dinâmicos**, **ícones (favicon + apple-icon + manifest)**, **páginas legais `/privacidade` e `/termos`**, e ativa o **ConsentBanner em produção** (já implementado na 1.2, aqui só validar integração + revisão jurídica das páginas legais).

Pré-requisito hard: ADR-005 do `architecture.md` corrigido para usar **e-mail `comercial@rfgcorretora.com.br`** e **founder "Ricardo Farias"** (não "Ricardo Guimarães") — CON-014, I-006, Dependência #12 PRD.

Referências: PRD FR-033/034/035/036/037/038/039/040, FR-046/047/048/049, FR-052; CON-004/014; ADR-005; `architecture.md` §4 + §7.

## Dev Notes

- **CON-014 / I-006 (Dependência #12):** correção do ADR-005 é **pré-flight hard**. Sem isso, JSON-LD vai pra produção com `contato@` e "Ricardo Guimarães" — bug compliance grave. Validar em PR.

- **`founder` no JSON-LD:** Ricardo Farias + Anderson Guimarães (CON-014, I-006). Importante para Knowledge Graph do Google.
- **Conteúdo `/privacidade` e `/termos`:** template padrão LGPD para SaaS/landing institucional brasileira aceito para v1 (Decisão Anderson 2026-05-05). Revisão jurídica externa pode ser feita em v1.1 sem bloquear go-live. Risco #4 do PRD descalonado.
- **OG image edge runtime:** `next/og` requer `runtime: 'edge'`. Não usa Node API (fs, etc.). Imagens externas via `fetch()` precisam ser `metadataBase` resolvíveis.
- **ConsentBanner em prod:** já está pronto da 1.2, aqui só ativar via env e validar fluxo end-to-end na URL Vercel Preview com `VERCEL_ENV=preview` (Pixel ainda NÃO carrega) e simular `production` localmente para confirmar carregamento gated.
- **Lighthouse SEO=100:** hard. Qualquer faltinha (alt em imagem, title curto, h1 ausente, lang errado, robots bloqueando) derruba o score. Auditar com cuidado.
